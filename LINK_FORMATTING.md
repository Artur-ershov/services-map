# Форматирование ссылок в описаниях

В описаниях сервисов (`desc` и `contacts`) можно использовать несколько способов для создания ссылок с читаемым текстом:

## 1. Markdown-подобный синтаксис (рекомендуется)

Используйте формат `[текст ссылки](URL)`:

```javascript
desc: "Больше информации: [Подробнее об EcoPoint](https://jive.croc.ru/docs/DOC-239136)"
```

**Результат:** Больше информации: [Подробнее об EcoPoint](https://jive.croc.ru/docs/DOC-239136)

## 2. Автоматическое определение из контекста

Если перед URL есть текст с двоеточием или в скобках, он автоматически станет текстом ссылки:

```javascript
desc: "Дополнительная информация: https://jive.croc.ru/community/knowledge/..."
```

**Результат:** Дополнительная информация: [Дополнительная информация](https://...)

```javascript
desc: "Подробнее (https://example.com/page)"
```

**Результат:** Подробнее: [Подробнее](https://example.com/page)

## 3. Обычные URL (автоматическое преобразование)

Если просто указать URL без контекста, он автоматически преобразуется в ссылку с текстом из домена:

```javascript
desc: "Сайт: https://example.com"
```

**Результат:** Сайт: [example.com](https://example.com)

## Примеры использования

### Пример 1: Markdown синтаксис
```javascript
desc: "Ознакомьтесь с [инструкциями по безопасности](https://jive.croc.ru/docs/SAFETY)"
```

### Пример 2: С контекстом
```javascript
desc: "Часы работы: с 9:00 до 18:00<br>Подробнее: https://jive.croc.ru/schedule"
```

### Пример 3: Несколько ссылок
```javascript
desc: "Документы: [Инструкция 1](https://example.com/doc1) и [Инструкция 2](https://example.com/doc2)"
```

## Важные замечания

- Все ссылки открываются в новой вкладке (`target="_blank"`)
- Если текст ссылки длиннее 50 символов, автоматически используется "Подробнее"
- Markdown-синтаксис имеет приоритет над автоматическим определением
- Можно комбинировать все три способа в одном описании
