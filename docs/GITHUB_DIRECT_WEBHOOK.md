# Прямая интеграция Tilda → GitHub Actions

## Проблема:

- Tilda отправляет **GET** запрос
- GitHub API для `repository_dispatch` требует **POST** с токеном
- Нужен промежуточный сервис

## Решения:

### Вариант 1: Vercel как прокси (текущий) ⭐

**Схема:**
```
Tilda (GET) → Vercel → GitHub API (POST) → GitHub Actions
```

**Преимущества:**
- ✅ Уже настроен
- ✅ Бесплатно
- ✅ Надежно

**Что нужно:** Добавить GitHub токен в Vercel

---

### Вариант 2: GitHub Webhook напрямую (сложнее)

GitHub Webhooks работают наоборот - они **отправляют** события, а не принимают.

Но можно использовать **GitHub App** или создать простой сервер.

---

### Вариант 3: Использовать готовый сервис

- **Zapier** (бесплатный план)
- **Make (Integromat)** (бесплатный план)
- **n8n** (self-hosted, бесплатно)

Но Vercel проще и уже настроен.

---

## Рекомендация:

**Оставьте Vercel как прокси** - это самое простое и надежное решение.

Просто добавьте GitHub токен в Vercel, и все будет работать автоматически.

## Альтернатива: Ручной триггер

Если не хотите настраивать автоматику, можно запускать синхронизацию вручную через GitHub UI:

1. GitHub → Actions → Tilda Sync on Webhook
2. Run workflow → Run workflow
3. Синхронизация запустится

Или через команду:
```bash
node scripts/download-tilda-files.js
```
